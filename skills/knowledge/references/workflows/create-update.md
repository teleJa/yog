# Create Or Update Workflow

Confirm scope before creating or updating a Candidate, Context, Capability, Evidence, ADR, or Business Flow. Route through the Knowledge indexes first, inspect the existing object and template, and preserve stable IDs and human-owned content.

Use the matching internal create script only after required fields and evidence are real. Do not create empty shells or placeholder business claims. Changes to formal boundaries, terminology, responsibilities, or status require explicit confirmation. Run sync and verify after writes.
