# Context Calibration Workflow

Collect overlap or boundary signals from Context responsibilities, non-responsibilities, terms, business-flow adjacency, duplicate reports, and semantic recall. Present evidence and options: keep separate with an explicit relationship, merge, split, rename, rewrite responsibilities, defer as `needs-review`, or gather more evidence.

Only the user may decide formal boundary, split/merge, ownership, relationship, terminology, or status changes. Until confirmed, remain read-only. After an authorized change, update all affected maps/flows and run sync and verify.
