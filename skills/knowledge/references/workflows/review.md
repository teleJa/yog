# Governance Review Workflow

Use this workflow when the user explicitly asks to inspect Candidate, `needs-review`, `stale`, or other low-confidence governance objects. These objects are excluded from `knowledge-query`, but may be read here for lifecycle review.

Default to a read-only inventory: paths, statuses, reasons, evidence gaps, duplicates, and recommended next decisions. Do not promote, edit, change status, rebuild indexes, or resolve an Audit finding unless the user separately authorizes that concrete action and its existing confirmation gate passes.
